# ForgeCLI Agent MVP PRD

## 1. 产品定位
ForgeCLI Agent MVP 是一个基于 Go 的本地终端代码助手。  
当前通过 OpenAI-compatible 协议接入 DeepSeek，重点验证“自然语言输入 -> 本地生成文件 -> 保存落地”的最小闭环。

## 2. 当前目标
在终端里实现一个最小可用 Agent，支持：

- 分析本地代码仓库
- 在受控模式下写文件
- 按用户需求生成单文件代码
- 将结果直接保存到本地

## 3. 用户使用场景

### 场景 1：分析本地项目
用户希望先了解一个本地项目的结构、文件内容和已有功能，因此进入只读模式查看和搜索代码。

### 场景 2：生成一个可直接使用的文件
用户希望通过自然语言快速生成一个目标文件，例如网页、脚本、配置文件或文档，并直接保存到本地。

### 场景 3：在终端中完成最小闭环
用户不希望只看到文本答案，而是希望 Agent 能真正把结果写成文件并落到目录中，形成可用结果。

## 4. 用户故事

### 用户故事 1
作为一个开发者，我希望在终端里直接查看本地项目结构和文件内容，这样我可以快速理解当前代码库。

### 用户故事 2
作为一个用户，我希望通过一句自然语言描述生成一个目标文件，并保存到指定路径，这样我不需要手动创建和粘贴代码。

### 用户故事 3
作为一个使用者，我希望系统在写文件和执行命令前先确认，这样我能确保本地环境不会被误操作。

### 用户故事 4
作为一个用户，我希望当模型调用失败时，系统仍然给出清晰反馈或最小可用兜底结果，这样我不会误以为任务完全失败。

## 5. 核心能力

### 5.1 Chat 模式
命令：

```powershell
forgecli chat --mode analyze
forgecli chat --mode full
```

支持多轮对话和基础命令：

- `/help`
- `/tools`
- `/mode analyze`
- `/mode full`
- `/reset`
- `/exit`

### 5.2 两种模式
`analyze`  
只读模式，可用于：

- 列文件
- 读文件
- 搜索内容

`full`  
可写模式，在只读能力基础上增加：

- 写文件
- 执行白名单命令
- 生成并保存文件

说明：  
`analyze` 模式下不会自动写文件，也不会自动切到 `full`，会提示用户手动切换。

### 5.3 通用单文件生成
命令：

```powershell
go run ./cmd/forgecli generate --prompt "..." --output file.ext
```

也支持在 `chat --mode full` 中直接说：

```text
帮我生成一个 python 脚本，保存成 tools/hi.py
```

当前生成逻辑：

- 用户提供需求
- 用户提供目标文件名
- Agent 根据目标文件扩展名生成完整文件
- 文件直接保存到本地

当前重点支持单文件生成，例如：

- `.html`
- `.py`
- `.go`
- `.js`
- `.ts`
- `.css`
- `.json`
- `.md`
- `.txt`

## 6. 安全机制

- 写文件前需要审批
- 执行命令前需要审批
- 限制写入路径在工作区内
- 敏感文件受保护
- 命令执行受白名单限制

## 7. 模型与兜底

### 模型接入
当前通过 OpenAI-compatible `/chat/completions` 协议接入 DeepSeek。

### 错误处理
- 超时自动重试一次
- 401 会提示检查 API Key
- 如果模型不可用，生成能力会使用本地最小 scaffold 兜底
- 如果文件已经成功写入，不会因为最终总结失败而误报整体失败

## 8. 当前范围
当前产品重点是“单文件生成闭环”，不是完整项目生成器，也不是完整自主工程 Agent。

## 9. 成功标准

当前阶段成功的标准是：

- 用户能在终端启动 Agent
- 用户能在 `analyze/full` 模式中切换
- 用户能分析本地项目
- 用户能按需求生成单文件代码并保存到本地
- Agent 能在异常情况下给出清晰反馈

## 10. 后续方向

- 支持多文件生成
- 减少对“必须明确写文件名”的依赖
- 提升通用代码生成质量
- 在闭环稳定后再考虑更完整的界面
